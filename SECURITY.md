# Security Policy

## Supported Versions

Security fixes are applied to the latest mainline version.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Preferred reporting channels:

1. GitHub Security Advisory (private report in this repository)
2. Direct maintainer contact via GitHub profile if advisory flow is unavailable

When reporting, include:

- affected component/file
- impact and attack scenario
- reproduction steps or proof of concept
- any suggested mitigation

## Response Expectations

- Initial acknowledgment target: within 72 hours
- Triage and severity classification: as soon as practical
- Fix timeline: based on severity and exploitability

## Disclosure

Please allow maintainers time to investigate and patch before public disclosure.
